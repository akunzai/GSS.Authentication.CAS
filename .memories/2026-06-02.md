#### 症狀
- Job: `build (React, samples/AspNetCoreReactSample, AspNetCoreReactSampleTests, https://localhost:5005)`
- Step 21 "Run E2E Tests" 執行約 5.5 分鐘後被手動取消

  - `Logout_ReturnsToAnonymousState`

  - `AuthenticatedUser_StaysLoggedIn_AfterNavigation`

#### 根本原因

**[Candidate] `Login.tsx` 的 `fetchSchemes()` 沒有 retry/error handling。** [13:20]

- `Login.tsx` 的 `fetchSchemes()` 呼叫 `/api/account/auth-schemes`
- 若請求失敗（任何原因），`schemes` 保持 `[]`，Login 頁面不渲染 CAS 按鈕
- React 的 `useEffect` 中 async 函數的 unhandled rejection 會被靜默忽略

**觸發條件（Vite 8 + CI 環境 race condition）：** [13:25]

- Vite 8 使用 Rolldown 打包器，第一次 module transformation 比 Vite 7 (esbuild) 慢
- Vite 8.0.15 新增 `send 408 on request timeout` 功能，改變 proxy 逾時行為
- CI 啟動流程：`curl` 確認 port 5005 就緒後立即啟動 E2E 測試，但 ASP.NET Core SPA middleware 透過 Vite proxy 代理請求，Vite 剛啟動時可能對 `/api/account/auth-schemes` 返回 408 或連線錯誤
- 此問題不是 PR #930（vite 8.0.14→8.0.16）造成的，而是既有問題在 Vite 8 下更容易觸發

#### 修復內容

**[Candidate] 在 `Login.tsx` 的 `fetchSchemes()` 加入 retry + backoff 邏輯。** [13:28]

- 最多 5 次重試，線性延遲（1s, 2s, 3s, 4s）
- 加入 `loading` 狀態，顯示 "Loading..." 提示
- tsc 和 ESLint 驗證均通過

### 本地調查與驗證結論 [13:38 - 13:50]

- **全新根本原因發現**：
  - 除了 Vite 剛啟動的 race condition，導致測試中 `ariaSnapshot` 呈空（頁面全空）的真正致命原因是 **React 版本不一致**！
  - Dependabot PR #929 升級了 `react-dom` 到 `19.2.7`，但沒有同步升級 `react`（維持在 `19.2.6`）。
  - React 19 要求 `react` 和 `react-dom` 必須有完全一致的版本，否則會在前端拋出 `Unhandled error: Incompatible React versions` 錯誤並直接 crash，導致 Playwright 無法取得任何渲染內容。
- **修復內容**：
  - 升級 `package.json` 中的 `react` 至 `19.2.7` 以保持一致。
  - 執行 `pnpm install --no-frozen-lockfile` 更新了 `pnpm-lock.yaml`。
  - 信任 macOS 本地 .NET HTTPS 開發憑證 (`dotnet dev-certs https --trust`)。
- **驗證結果**：
  - 本地啟動 `AspNetCoreReactSample` (port 5005) 及 Docker Keycloak & Redis 後，執行 `dotnet test --filter "FullyQualifiedName~AspNetCoreReactSampleTests"`，**所有 9 個 React E2E 測試全部順利通過**！
